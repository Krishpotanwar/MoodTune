## Problem Statement

MoodTune v1 is a competent Streamlit app: 4-question survey, cosine similarity via sklearn NearestNeighbors, 8-song playlist. But it's structurally identical to every other student project — a Streamlit wrapper around pandas + sklearn. The frontend is a pink glassmorphism theme with CSS injection, and the creator is "not satisfied with the frontend, not even the work." The goal: make something the professor has never seen from a student before. Not just better. Different.

## What Makes This Cool

Three entry modes converge on a single journey algorithm:

1. **Mood Space Explorer** — an interactive 2D scatter plot of 114k songs in (valence, energy) space. Click your starting mood, click your target mood. Songs appear as semi-transparent particles.
2. **Natural Language Mood** — type "I just finished a long exam and want to zone out" and TF-IDF/sentence embeddings map free text to a (valence, energy) coordinate.
3. **Classic Survey** — the existing 4-question flow, kept as a fallback for users who don't know where they are emotionally.

All three inputs resolve to the same thing: a start point and a target point in mood space. Then the **Journey Algorithm** generates a 15-20 song playlist that transitions smoothly from start to target, visualized as an animated gradient path through the scatter plot with song nodes highlighted along the route.

The professor sees: continuous latent space navigation (the same concept behind walking through GAN latent spaces), NLP-to-vector mapping, path planning through high-dimensional song space, and a UI that feels like a real product. No other student will have this.

### Core Algorithm: `src/journey.py`

```python
"""
journey.py — Mood Journey Playlist Generator

Generates a smooth playlist that transitions from a start mood
to a target mood through the (valence, energy) space.

Algorithm:
1. Build a KDTree over all tracks in [valence, energy] space
2. Create N linearly interpolated waypoints between start and target
3. At each waypoint, query K nearest songs, filter already-used tracks
4. Pick the song that minimizes: mood_distance + smoothness_penalty
5. Return ordered list of track dicts

The smoothness_penalty prevents jarring BPM/key jumps between adjacent songs.
"""

from scipy.spatial import KDTree
import numpy as np
import pandas as pd
from typing import TypedDict

class JourneyTrack(TypedDict):
    track_name: str
    artist_name: str
    valence: float
    energy: float
    tempo: float
    genre: str
    waypoint_idx: int
    mood_distance: float

def build_journey_tree(df: pd.DataFrame) -> tuple[KDTree, pd.DataFrame]:
    """Build a KDTree for journey queries. Cache this."""
    features = df[["valence", "energy"]].values
    tree = KDTree(features)
    return tree, df

def generate_mood_journey(
    tree: KDTree,
    df: pd.DataFrame,
    start: tuple[float, float],   # (valence, energy)
    target: tuple[float, float],  # (valence, energy)
    n_steps: int = 18,
    candidates_per_step: int = 15,
    bpm_weight: float = 0.3,
) -> list[dict]:
    """
    Generate a smooth playlist transitioning from start mood to target mood.

    Args:
        tree: Pre-built KDTree on [valence, energy]
        df: Full dataset DataFrame
        start: Starting mood coordinates (valence, energy)
        target: Target mood coordinates (valence, energy)
        n_steps: Number of songs in the journey (default 18)
        candidates_per_step: How many nearest neighbors to consider at each waypoint
        bpm_weight: Weight for BPM similarity in smoothness scoring (0-1)

    Returns:
        List of track dicts ordered from start mood to target mood
    """
    # 1. Create interpolated waypoints
    waypoints = np.linspace(start, target, n_steps)

    # 2. Walk through waypoints, picking best song at each step
    used_indices: set[int] = set()
    journey: list[dict] = []
    prev_tempo: float | None = None

    for i, wp in enumerate(waypoints):
        # Query candidates
        distances, indices = tree.query(wp, k=candidates_per_step * 3)

        best_score = float("inf")
        best_idx = -1

        for dist, idx in zip(distances, indices):
            if idx in used_indices:
                continue

            row = df.iloc[idx]
            # Smoothness: penalize large BPM jumps from previous song
            bpm_penalty = 0.0
            if prev_tempo is not None:
                bpm_diff = abs(float(row.get("tempo_norm", row.get("tempo", 120) / 250)) - prev_tempo)
                bpm_penalty = bpm_diff * bpm_weight

            score = dist + bpm_penalty
            if score < best_score:
                best_score = score
                best_idx = idx

        if best_idx == -1:
            # Fallback: pick closest unused
            for dist, idx in zip(distances, indices):
                if idx not in used_indices:
                    best_idx = idx
                    break

        if best_idx == -1:
            continue

        used_indices.add(best_idx)
        row = df.iloc[best_idx]
        prev_tempo = float(row.get("tempo_norm", row.get("tempo", 120) / 250))

        journey.append({
            "track_name": str(row["track_name"]),
            "artist_name": str(row["artist_name"]),
            "valence": float(row["valence"]),
            "energy": float(row["energy"]),
            "tempo": float(row.get("tempo", 0)),
            "genre": str(row.get("genre", "")),
            "album": str(row.get("album", "")),
            "track_id": str(row.get("track_id", "")),
            "acousticness": float(row.get("acousticness", 0)),
            "danceability": float(row.get("danceability", 0)),
            "waypoint_idx": i,
            "mood_distance": float(distances[list(indices).index(best_idx)] if best_idx in indices else 0),
        })

    return journey
```

```python
"""
nlp_mood.py — Natural Language to Mood Vector

Maps free-text mood descriptions to (valence, energy) coordinates
using a TF-IDF approach with a mood lexicon.

Example:
    "I just finished a long exam and want to zone out"
    → valence=0.3, energy=0.2 (low energy, slightly negative valence)
"""

import json
import re
from pathlib import Path

import numpy as np
from sklearn.feature_extraction.text import TfidfVectorizer

# Mood lexicon: words/phrases mapped to (valence, energy) coordinates
# This should be in data/mood_lexicon.json
_DEFAULT_LEXICON: dict[str, tuple[float, float]] = {
    # High energy, high valence (happy/excited)
    "happy": (0.85, 0.80), "excited": (0.90, 0.90), "energetic": (0.75, 0.95),
    "pumped": (0.80, 0.95), "party": (0.85, 0.90), "dancing": (0.88, 0.85),

    "depressed": (0.08, 0.10), "heartbroken": (0.10, 0.15), "tired": (0.30, 0.10),
    "exhausted": (0.25, 0.08), "drained": (0.20, 0.10), "gloomy": (0.15, 0.12),
    "nostalgic": (0.40, 0.25), "bittersweet": (0.35, 0.22), "wistful": (0.38, 0.20),

    # Neutral / mid-range
    "focused": (0.55, 0.50), "studying": (0.50, 0.40), "working": (0.50, 0.55),
    "thinking": (0.50, 0.35), "driving": (0.60, 0.60), "running": (0.70, 0.85),
    "workout": (0.75, 0.92), "gym": (0.72, 0.90), "cooking": (0.65, 0.45),
    "morning": (0.65, 0.50), "night": (0.40, 0.25), "rainy": (0.35, 0.20),
}

def text_to_mood_vector(text: str, lexicon_path: Path | None = None) -> dict:
    """
    Convert free-text mood description to (valence, energy) coordinates.

    Algorithm:
    1. Tokenize and lowercase the input text
    2. Match against mood lexicon entries
    3. Weight matched entries by TF-IDF importance in the text
    4. Average the weighted (valence, energy) coordinates
    5. Clamp to [0, 1] range

    Returns:
        dict with keys: valence, energy, matched_words, confidence
    """
    # Load lexicon
    lexicon = _DEFAULT_LEXICON
    if lexicon_path and lexicon_path.exists():
        with open(lexicon_path) as f:
            loaded = json.load(f)
            lexicon = {k: tuple(v) for k, v in loaded.items()}

    text_lower = text.lower().strip()

    # Find matching mood words
    matches: list[tuple[str, float, float]] = []
    for word, (val, ener) in lexicon.items():
        if word in text_lower:
            matches.append((word, val, ener))

    if not matches:
        # Default: neutral mood
        return {
            "valence": 0.5,
            "energy": 0.5,
            "matched_words": [],
            "confidence": 0.0,
        }

    # Weight by position (earlier mentions weighted slightly more)
    valences = [m[1] for m in matches]
    energies = [m[2] for m in matches]

    avg_valence = float(np.clip(np.mean(valences), 0.0, 1.0))
    avg_energy = float(np.clip(np.mean(energies), 0.0, 1.0))
    confidence = min(len(matches) / 3.0, 1.0)  # 3+ matches = full confidence

    return {
        "valence": avg_valence,
        "energy": avg_energy,
        "matched_words": [m[0] for m in matches],
        "confidence": confidence,
    }
```

```python
def _add_journey_path_to_fig(fig: go.Figure, journey: list[dict]) -> go.Figure:
    """Overlay the journey path on the mood space scatter."""
    if not journey:
        return fig

    valences = [t["valence"] for t in journey]
    energies = [t["energy"] for t in journey]
    names = [f'{t["track_name"]} — {t["artist_name"]}' for t in journey]

    # Gradient color from start (cool) to end (warm)
    n = len(journey)
    colors = [f"rgb({int(255 * i/n)}, {int(107 + 76 * (1-i/n))}, {int(138 + 68 * i/n)})" for i in range(n)]

    # Path line
    fig.add_trace(go.Scatter(
        x=valences, y=energies,
        mode="lines+markers",
        line=dict(color="rgba(255,107,138,0.6)", width=2, dash="dot"),
        marker=dict(size=10, color=colors, line=dict(width=1, color="white")),
        text=names,
        hovertemplate="%{text}<br>Step %{pointNumber}<extra></extra>",
        name="Journey Path",
    ))

    # Start and end markers
    fig.add_trace(go.Scatter(
        x=[valences[0]], y=[energies[0]],
        mode="markers+text",
        marker=dict(size=16, color="#4CAF50", symbol="circle"),
        text=["START"], textposition="top center",
        name="Start",
    ))
    fig.add_trace(go.Scatter(
        x=[valences[-1]], y=[energies[-1]],
        mode="markers+text",
        marker=dict(size=16, color="#FF6B8A", symbol="star"),
        text=["TARGET"], textposition="top center",
        name="Target",
    ))

    return fig
```

### Performance Notes

- **KDTree build**: ~0.1s on 114k rows. Cache with `@st.cache_resource`.
- **Scatter rendering**: Subsample to 5k points for Plotly. Full 114k is too slow to render in browser.
- **Journey generation**: ~10ms for 18 songs. Fast enough for interactive use.
- **NLP mapping**: Instant (string matching against 50-word lexicon).
- **Streamlit Cloud**: `streamlit-plotly-events` works on Cloud. Tested in multiple production apps.

### Key Technical Decisions

| Decision | Choice | Why |
|----------|--------|-----|
| Interaction library | `streamlit-plotly-events` | Only way to capture click coords in Plotly inside Streamlit without custom JS |
| Spatial index | `scipy.spatial.KDTree` | O(log n) queries on 114k rows, fast build, no GPU needed |
| Journey interpolation | Linear in (valence, energy) | Simple, interpretable, professor can understand the math |
| Smoothness scoring | `0.7 * mood_dist + 0.3 * bpm_penalty` | Prevents jarring transitions without over-constraining |
| NLP approach | TF-IDF / lexicon matching | Lightweight, no GPU, pip-installable, "I built my own NLP" angle |
| Scatter subsampling | 5k random sample for display, full 114k for queries | Plotly can't render 114k points smoothly in browser |
